# ItemChecklist UI Pivot — Iteration 1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the broken uGUI window with a minimal SpriteRenderer-based window using CoreLib's `UserInterfaceModule` — proves the architecture before scoping a full feature port.

**Architecture:** Cross-repo: add CoreLib git package to shared `CoreKeeperModSDK/Packages/manifest.json`. In this mod: declare CoreLib in asmdef + ModBuilderSettings, write a new `IModUI`-implementing window class extending `UIelement`, replace the uGUI prefab with a SpriteRenderer-based skelett that the user finishes in the Unity Editor. Old `ui/` files remain in tree (disconnected from runtime) as Iter-2 reference.

**Tech Stack:** Unity 6000.0.59f2 SDK, C# 9 (Roslyn under Pugstorm sandbox), CoreLib 4.0.4, Harmony 2.x (via PugMod), CoreKeeperModSDK build pipeline.

### Task 5: Create ItemChecklistWindow.cs

**Files:**

- [ ] **Step 1: Write the script file**

Create `unity/ItemChecklist/ui/ItemChecklistWindow.cs`:

```csharp
using CoreLib.Submodule.UserInterface.Interface;
using UnityEngine;

namespace ItemChecklist.UI
{
    public class ItemChecklistWindow : UIelement, IModUI
    {
        // Editor-wired serialized fields
        public GameObject root;
        public SpriteRenderer background;
        public PugText title;

        // IModUI implementation
        public GameObject Root => root;
        public bool ShowWithPlayerInventory => false;
        public bool ShouldPlayerCraftingShow => false;

        protected void Awake()
        {
            HideUI();
        }

        public void ShowUI()
        {
            root.SetActive(true);
            ApplyTheme();
        }

        public void HideUI()
        {
            root.SetActive(false);
        }

        private void ApplyTheme()
        {
            // Vanilla CK CraftingUI theme as 9-slice background.
            // GetCraftingUITheme takes UIManager.CraftingUIThemeType enum
            // (verified from BookMod/Scripts/UI/BookUI.cs:71). Wood is an
            // educated guess from naming convention; if invalid, fallback
            // path loads our own atlas sprite.
            try
            {
                var theme = Manager.ui.GetCraftingUITheme(UIManager.CraftingUIThemeType.Wood);
                if (theme != null && background != null)
                    background.sprite = theme.background;
            }
            catch (System.Exception ex)
            {
                Debug.LogWarning($"[ItemChecklist] GetCraftingUITheme failed: {ex.Message} — falling back to atlas sprite");
                if (background != null && ItemChecklistMod.AssetBundle != null)
                    background.sprite = ItemChecklistMod.AssetBundle.LoadAsset<Sprite>("ui_panel");
            }

            if (title != null)
                title.Render("Item Checklist");
        }
    }
}
```

Show/Hide/Awake pattern is 1:1 from `/tmp/ck-ui-research/limoka-CoreKeeperMods/SDK Mods/Assets/Mods/DummyMod/Scripts/UI/DummyUI.cs`.
